记一次网站升级过程

今天是最后一天 我完成了一个网站的服务器软件的升级。服务器是CentOS6.5 升级目标是

apache 2.2.15-> apache 2.4.10 php5.3->php5.6 mysql5.0->mysql5.1

本来想着挺简单的 但是没想到遇到了很多问题 记录下来 以便以后参考

最开始我用putty连上去(一个Windows下的ssh客户端)，结果发现那个服务器没有联网，yum就不能用了，而且连个gcc都没有。

于是我先把yum的yum.conf设置了代理到我的电脑上 注意关闭下自己电脑的防火墙

vi /etc/yum.conf

添加一行

proxy=http://10.128.50.121:8080

这就需要给我的机器上架设一个代理服务器，刚开始想简单就用自由门，发现速度太慢了。。

后来使用了ccproxy(http://www.ccproxy.com/)还是挺不错的，可以使用ip统配匹配。其实http proxy其实自己写一个也不难，就是可靠性不知道

代理服务器架设完后 ，这yum update 就能解决问题了吗？先不管装个gcc再说，一切顺利要是自己一个个找rpm还不麻烦死

yum install gcc

然后就想着 yum update php连同httpd一起更新就好了，但是发现它已经是最新版本了。

我换了网易的centos yum 源，结果还是一样，分析了下原因，是它的操作系统版本路径就限定了是基于centos 6的软件，

然后我就自作聪明的把CentOS-Base.repo(/etc/yum)里面的$releasever改成了7 这样它应该就会从centos7的仓库下载了，

baseurl=http://mirrors.163.com/centos/$releasever/os/$basearch/

在yum info 的时候一切顺利 可以得到Apache 2.4的rpm信息 但是在装的时候却出问题了。

有几个依赖项一直循环依赖 导致yum install一直就卡在那里了 只好使用 kill -9 pid 杀进程了。

多次尝试之后无果 最后只好放弃了

既然yum不给力 就只能选择源码编译构建了

我先装apache。从网上下载到apache及其依赖的包apr apr-util pcre

过程如下

安装apr
tar -zxvf apr.tar.gz
    cd apr
    ./configure --prefix=/usr/local/apr
    make
    make install
安装apr-util
tar -zxvf apr-util.tar.gz
    cd apr-util
    ./configure --prefix=/usr/local/apr-util --with-apr=/usr/local/apr
    make
    make install
安装pcre
./configure --prefix=/usr/local/pcre
    make
    make install
安装apache
./configure --prefix=/usr/local/apache --with-apr-util=/usr/local/apr-util/ --with-pcre=/usr/local/pcre/
    --with-apr=/usr/local/apr/
    make
    make install

然后是php。php依赖libxml。故需要先装libxml2 过程不再详述

然后安装php 注意--with-apxs2指向apache --with-libxml-dir指向libxml

./configure --prefix=/usr/local/php --with-apxs2=/usr/local/apache/bin/apxs --with-libxml-dir=/usr/local/libxml2 --with-mysql=mysqlnd --with-pdo-mysql=mysqlnd

注意这个

--with-mysql=mysqlnd --with-pdo-mysql=mysqlnd

我当时没有加，后来发现数据库无法访问 然后我就检查了下，couldn't find MySQL driver 。

这个需要在安装PHP的时候就注意下。还有php.ini的操作就不说了

extension=pdo_mysql.so
    mysql.default_socket = /var/lib/mysql/mysql.sock
    pdo_mysql.default_socket= /var/lib/mysql/mysql.sock
    mysqli.default_socket =/var/lib/mysql/mysql.sock

编辑 /usr/local/apache/conf/httpd.conf 支持php AddType application/x-httpd-php .php

/usr/local/apache/bin/apachectl start

启动apache

然后我突然觉得发现不知道怎么就把数据库不小心卸载了， 然后只好重装一遍，所幸它不删除数据。

重装就用 yum 就可以了，yum install mysql & yum install mysql-server 客户端和服务端

然后发现InnoDB 没有启用报错是 Unknown table engine 'InnoDB'

于是进入mysql 执行show engines; 发现就没有innodb 引擎

安装innodb插件

INSTALL PLUGIN INNODB SONAME 'ha_innodb_plugin.so';

中间又出现了些问题mysqld已死 但是...等异常信息，注意查看 /var/log/mysql.log 比较简单的方案就是

ps aux|grep mysql

找出和mysql相关的进程 全部kill之后再启动 service mysqld start 注意提升权限，我一般都是

chmod 777 xxx.xx

service mysqld start

最后mysql也可以运行了

这样，我的网站也就重新可以访问了，也是费了好长时间，还是经验太少导致的。

关于服务器的防火墙

vi /etc/sysconfig/iptables

我配置开启22 80 端口 有时候还需要使用3306远程访问数据库

 # Firewall configuration written by system-config-firewall
    # Manual customization of this file is not recommended.
    *filter
    :INPUT ACCEPT [0:0]
    :FORWARD ACCEPT [0:0]
    :OUTPUT ACCEPT [0:0]
    -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
    -A INPUT -p icmp -j ACCEPT
    -A INPUT -i lo -j ACCEPT
    -A INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT
    -A INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT
    #-A INPUT -m state --state NEW -m tcp -p tcp --dport 3306 -j ACCEPT
    -A INPUT -j REJECT --reject-with icmp-host-prohibited
    -A FORWARD -j REJECT --reject-with icmp-host-prohibited
    COMMIT

一些防火墙操作命令

service iptables stop service iptables start service iptables status service iptables restart

备份下数据库吧

cd /var/lib/mysql

mysqldump --lock-tables=false -u user -p db_name > xxx.sql.bak 然后回车输入密码
